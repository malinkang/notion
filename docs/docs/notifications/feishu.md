# 飞书通知配置

NotionHub 可以通过飞书群机器人发送同步通知。飞书通知通常使用群机器人的 Webhook 地址。

配置飞书通知前，需要先准备：

- Webhook 地址
- 签名密钥。如果开启了签名校验，需要填写

## 一、如何申请

### 1. 创建飞书群机器人

在飞书中打开用于接收通知的群聊。

进入群设置，找到 `群机器人`，添加 `自定义机器人`。

填写机器人名称，例如：

```text
NotionHub 通知
```

### 2. 配置安全设置

飞书自定义机器人通常可以配置安全策略，常见方式包括：

- 自定义关键词
- 签名校验
- IP 白名单

建议先使用 `自定义关键词`，例如填写：

```text
NotionHub
```

如果你选择 `签名校验`，创建机器人后需要同时保存：

- Webhook 地址
- 签名密钥

### 3. 复制 Webhook

机器人创建完成后，飞书会生成一个 Webhook 地址，格式大致如下：

```text
https://open.feishu.cn/open-apis/bot/v2/hook/xxxx
```

请保存这个地址，后续需要填写到 NotionHub 插件中。

## 二、如何在插件中配置

1. 打开 NotionHub 浏览器插件。
2. 进入需要接收通知的同步服务。
3. 找到通知配置区域。
4. 通知平台选择 `飞书`。
5. 填写飞书机器人 Webhook 地址。
6. 如果插件提供签名密钥输入框，并且你在飞书中开启了签名校验，请填写签名密钥。
7. 点击保存或提交。
8. 运行一次同步，检查飞书群是否收到通知。

## 常见问题

如果收不到通知，优先检查：

- Webhook 地址是否复制完整。
- 飞书安全设置是否和插件配置一致。
- 如果使用关键词，通知内容是否包含对应关键词。
- 如果使用签名校验，签名密钥是否填写正确。
